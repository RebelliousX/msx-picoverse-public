# MSX PicoVerse 2350 Dual PSG Implementation

This document describes the Dual PSG implementation used by PicoVerse 2350 firmware. It covers the common secondary PSG sound engine, the LoadROM command-line option, the Explorer per-ROM audio profile, and how Explorer's independent primary **PSG** mirror differs from Dual PSG.

## Terminology

Dual PSG adds a second AY-3-8910 compatible PSG beside the MSX internal PSG. The second PSG is accessed through I/O ports `0x10` and `0x11` and produces audio through the cartridge output. This term is used in the firmware, Explorer menu, and documentation.

Explorer also has a separate ROM-detail **PSG** Yes/No option. That option mirrors the primary MSX PSG ports `0xA0` and `0xA1` to the cartridge DAC; it is not the same thing as Dual PSG and does not add a second PSG port range.

## User-facing behavior

The MSX internal PSG remains active and unchanged. PicoVerse Dual PSG adds a second, cartridge-side PSG for software that explicitly writes to the secondary PSG ports.

The feature is available in two PicoVerse 2350 workflows:

- **LoadROM**: enabled with `loadrom.exe -d` or `loadrom.exe --dual-psg` when building a single-ROM UF2.
- **Explorer**: selected from the ROM detail screen as the **Dual PSG** audio profile for supported ROM entries.

Dual PSG is intended for homebrew, music players, demos, or patched games that know how to use a second PSG. A normal PSG-only game will continue using the internal MSX PSG and will not automatically gain extra channels unless the software writes to ports `0x10` and `0x11`. In Explorer, a normal PSG-only game can still be mirrored through the cartridge DAC by setting the separate **PSG** field to **Yes** on the ROM detail screen.

## Compatibility rules

PicoVerse treats cartridge audio profiles as a single mutually exclusive mode per ROM. A ROM may run with no extra cartridge audio, SCC, SCC+, Dual PSG, or MSX-MUSIC, but not more than one of those profiles at the same time.

Explorer's primary **PSG** mirror is independent from that profile choice. It can be enabled alongside None, SCC, SCC+, Dual PSG, or MSX-MUSIC for regular ROM launches.

Dual PSG is allowed for regular ROM mappers that do not already reserve the cartridge audio slot for another sound engine.

Dual PSG is not available for:

- Konami SCC mapper ROMs.
- Manbow2 mapper ROMs.
- SCC or SCC+ audio modes.
- Sunrise / Nextor system modes.
- Carnivore2 system modes in LoadROM.
- Explorer SYSTEM entries, File Hunter folders, and folder records.

Konami SCC and Manbow2 ROMs are excluded because those cartridges carry or expect SCC audio. In PicoVerse 2350, that audio path is reserved for SCC/SCC+ emulation, so the second PSG option is not offered or accepted for those mapper types.

## MSX I/O port model

The secondary PSG follows the common MSX expansion convention used by devices such as Carnivore2, MegaFlashROM SCC+ SD, FlashJacks, and GR8NET:

| Port | Direction | Meaning |
| --- | --- | --- |
| `0x10` | Write | Secondary PSG register select |
| `0x11` | Write | Secondary PSG register data |

The PicoVerse implementation focuses on the write path used by music engines. Reads from the secondary PSG ports are not used for audio generation and are answered with an inactive `0xFF` response when the firmware sees an I/O read cycle.

Example Z80 flow:

```asm
ld a,8          ; PSG channel A volume register
out (0x10),a    ; select register 8 on the second PSG
ld a,15         ; max volume
out (0x11),a    ; write value to selected register
```

## PSG emulation engine

The firmware uses the `emu2149` PSG emulator to model the secondary AY/YM-compatible chip.

Shared audio constants:

| Constant | Value | Purpose |
| --- | --- | --- |
| `PSG_SAMPLE_RATE` | `44100` | I2S output sample rate |
| `PSG_CLOCK` | `1789773` | PSG clock, matching the MSX PSG half-clock convention |
| `PSG_VOLUME_SHIFT` | `2` | Digital gain applied before clamping |
| `PSG_PORT_REG` | `0x10` | Register select port |
| `PSG_PORT_DATA` | `0x11` | Data write port |

Initialization performs these steps:

1. Clear the static `PSG` instance.
2. Set the emulator output rate to 44100 Hz.
3. Select volume mode `2`.
4. Set the PSG clock to 1,789,773 Hz.
5. Enable quality mode with `PSG_setQuality(..., 1)`.
6. Reset the PSG registers.
7. Initialize the PIO1 I/O bus captors.
8. Start I2S audio output when the selected audio mode requires Dual PSG.

Each audio sample is produced with `PSG_calc()`, shifted left by `PSG_VOLUME_SHIFT`, clamped to signed 16-bit range, and written to both left and right I2S channels.

## Audio output path

Dual PSG audio is generated by the cartridge and sent to the RP2350 I2S DAC path. The cartridge-side PSG output is external audio, just like SCC/SCC+ output.

When Explorer's separate **PSG** option is enabled, primary PSG writes on `0xA0`/`0xA1` are also captured and mixed into the DAC stream. The physical MSX PSG still sounds normally; PicoVerse provides a DAC-side mirror rather than replacing the internal chip.

The firmware uses:

- Stereo 16-bit PCM audio buffers.
- 44.1 kHz sample rate.
- 256 samples per buffer.
- A producer pool with 3 buffers.
- The I2S DAC pin set already used by the 2350 audio path:
  - Data: GPIO 29.
  - Bit clock base: GPIO 30.
  - Word select: GPIO 31.
  - Mute control: GPIO 32.

The same mono PSG sample is written to both stereo channels.

## PIO bus capture

PicoVerse 2350 already uses PIO state machines for the MSX cartridge bus. Dual PSG adds use of the I/O bus captors so the firmware can see Z80 I/O cycles while the ROM mapper hot loop continues serving memory reads and writes.

The I/O bus layer captures:

- I/O writes: address plus data byte.
- I/O reads: address, followed by a firmware response token.

For Dual PSG writes, the firmware masks the captured address to the low 8 bits and handles only ports `0x10` and `0x11`:

```c
uint8_t port = io_addr & 0xFFu;
if (port == PSG_PORT_REG || port == PSG_PORT_DATA) {
    PSG_writeIO(&psg_instance, port, io_data);
}
```

Other I/O writes are ignored by the Dual PSG service routine. I/O reads are answered as not-in-window with `0xFF` so the MSX bus is released cleanly.

A timing fix was added on the LoadROM side so the PIO1 I/O write captor samples while `/WR` is low after the address and data lines have settled. This makes plain `OUT (0x10),A` and `OUT (0x11),A` writes reliable.

## Core usage

Dual PSG has two jobs that must keep running while the game executes:

1. Consume I/O writes to ports `0x10` and `0x11`.
2. Generate continuous I2S audio samples.

The current LoadROM implementation moves Dual PSG I/O servicing to the audio producer core. This avoids depending on cartridge memory read cycles to drain PSG writes. It matters for ROMs or modes that run from RAM or spend long periods without cartridge reads.

Explorer follows the same idea: when the resolved audio mode is Dual PSG, it initializes the PSG, starts the I/O captor, initializes I2S, and launches a core 1 audio loop that repeatedly services I/O and fills audio buffers.

## LoadROM implementation

LoadROM exposes Dual PSG through a command-line option:

```text
-d, --dual-psg
```

The tool validates the requested audio mode before writing the UF2:

- `-d` cannot be combined with `-scc` or `-sccplus`.
- `-d` requires a normal external ROM file.
- `-d` is rejected for Konami SCC and Manbow2 mappers.
- `-d` is rejected for embedded Sunrise / Carnivore2 system modes.

When accepted, the tool sets bit `0x10` in the configuration `rom_type` byte:

| Bit | Meaning |
| --- | --- |
| `0x80` | SCC enabled |
| `0x40` | SCC+ enabled |
| `0x20` | WiFi support for Sunrise modes |
| `0x10` | Dual PSG enabled |
| low bits | Base mapper ID |

At firmware startup, LoadROM strips the audio and WiFi flags to recover the base mapper, then resolves the requested cartridge audio mode:

```c
AUDIO_MODE_NONE
AUDIO_MODE_SCC
AUDIO_MODE_SCC_PLUS
AUDIO_MODE_DUAL_PSG
```

The resolver enforces system-mode and SCC-capable mapper restrictions. If it returns `AUDIO_MODE_DUAL_PSG`, the firmware enables the Dual PSG engine before dispatching to the selected mapper loop.

- `2350/software/loadrom.pio/tool/src/loadrom.c`
- `2350/software/loadrom.pio/pico/loadrom/loadrom.h`
- `2350/software/loadrom.pio/pico/loadrom/loadrom.c`
- `2350/software/loadrom.pio/pico/loadrom/emu2149.c`
- `2350/software/loadrom.pio/pico/loadrom/emu2149.h`

## Explorer implementation

Explorer exposes the same sound engine as a per-ROM audio profile in the MSX menu.

The ROM detail screen supports these profiles:

- `None`
- `SCC`
- `SCC+`
- `Dual PSG`
- `MSX-MUSIC`

The ROM detail screen also has an independent **PSG** Yes/No field. It defaults to **No** and can be enabled to mirror the primary MSX PSG through the cartridge DAC while any valid Audio profile is selected.

The menu only cycles through profiles valid for the selected record:

- Konami SCC and Manbow2 ROMs can use `None`, `SCC`, or `SCC+`.
- Regular non-SYSTEM, non-SCC ROMs can use `None`, `Dual PSG`, or `MSX-MUSIC`.
- SYSTEM entries and folders do not offer Dual PSG.

When the user selects **Action: Run**, the MSX menu writes the selected audio profile to `CTRL_AUDIO` in the Explorer control window. The Pico firmware stores that byte as `ctrl_audio_selection`.

Explorer firmware resolves that byte into an internal audio mode:

```c
AUDIO_MODE_NONE
AUDIO_MODE_SCC
AUDIO_MODE_SCC_PLUS
AUDIO_MODE_DUAL_PSG
AUDIO_MODE_MSX_MUSIC
```

If the resolved mode is `AUDIO_MODE_DUAL_PSG`, Explorer starts the Dual PSG audio engine before dispatching to the ROM mapper handler.

If the separate **PSG** field is enabled, Explorer also initializes a primary PSG emulator for ports `0xA0`/`0xA1`. With SCC, SCC+, Dual PSG, or MSX-MUSIC selected, the primary PSG mirror is serviced by the same core 1 audio loop and mixed into the same I2S stream.

Key Explorer files:

- `2350/software/explorer.pio/msx/src/menu.h`
- `2350/software/explorer.pio/msx/src/screen_rom.c`
- `2350/software/explorer.pio/pico/explorer/explorer.c`
- `2350/software/explorer.pio/pico/explorer/emu2149.c`
- `2350/software/explorer.pio/pico/explorer/emu2149.h`

## Relationship with SCC/SCC+

SCC/SCC+ and Dual PSG are both cartridge-side audio engines, but they model different expansion hardware:

| Audio profile | Intended ROMs | Ports / access model |
| --- | --- | --- |
| SCC | Konami SCC / Manbow2 | SCC memory-mapped register behavior inside SCC mapper windows |
| SCC+ | Konami SCC+ / SCC-I compatible software | SCC+ enhanced register behavior |
| Dual PSG | ROMs or patches written for second PSG hardware | I/O ports `0x10` and `0x11` |
| MSX-MUSIC | ROMs that can use YM2413 / FM-PAC audio | I/O ports `0x7C` and `0x7D` |

A ROM should use only one cartridge audio profile at a time. For this reason, both the LoadROM tool and Explorer UI prevent invalid profile combinations instead of letting incompatible audio engines fight for core 1, I2S, or cartridge audio semantics. Explorer's primary **PSG** mirror is a separate mixer input rather than a cartridge audio profile.

## Software requirements on the MSX side

Dual PSG does not patch or reinterpret game audio. The MSX software must already support a second PSG. Compatible software usually does one of the following:

- Writes music data directly to ports `0x10` and `0x11`.
- Provides a configuration option for Carnivore2 / MegaFlashROM / FlashJacks style second PSG hardware.
- Uses a patched soundtrack driver that splits music across the internal PSG and the secondary PSG.

The primary PSG remains at the normal MSX ports (`0xA0`, `0xA1`, and optionally `0xA2`). Dual PSG adds the second chip only. To mirror normal PSG-only software through the cartridge DAC in Explorer, use the ROM detail **PSG** option instead of the **Dual PSG** audio profile.

## Testing checklist

A basic validation path is:

1. Use a ROM, demo, or music player known to support a second PSG on ports `0x10` and `0x11`.
2. For LoadROM, build with `loadrom.exe -d romfile.rom`.
3. For Explorer, open the ROM detail screen and select **Dual PSG**.
4. Optionally set **PSG: Yes** if you also want the primary PSG mirrored through the cartridge DAC.
5. Confirm the ROM is not Konami SCC, Manbow2, or a SYSTEM entry.
6. Run the title and listen for the extra PSG voices through the cartridge audio output.
7. Compare against `None` to confirm the additional channels disappear when the profile is disabled.

For low-level testing, a small Z80 program can write a tone period and volume to the secondary PSG through ports `0x10` and `0x11` and verify that only the cartridge audio path produces the extra tone.

## Known limitations

- Readback from the secondary PSG is not implemented as a meaningful PSG register read path; reads are answered inactive.
- Software must explicitly use the second PSG ports. Existing PSG-only software will not automatically become six-channel audio.
- Dual PSG is mutually exclusive with SCC/SCC+ and MSX-MUSIC as a cartridge audio profile.
- Dual PSG is not available in embedded storage/system modes that already need the second core or have different cartridge behavior.
- LoadROM Dual PSG outputs the emulated second PSG through the cartridge I2S audio path; it does not digitally mix or emulate the MSX internal PSG. Explorer can mirror the primary PSG only when the separate **PSG** ROM detail option is enabled.
